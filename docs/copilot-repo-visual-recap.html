Section 5

Data Integrity & Normalization Rules

Integrity controls are taken from mapping quality gates and normalization policy, without adding schema-only checks.

Quality Gate Check
no_orphan_taskseach oper_task must link to exactly one project
no_orphan_voice_messageseach voice_message must link to one voice_session
no_orphan_voice_history_stepseach history_step must link to one voice_session
no_orphan_voice_merge_logseach voice_session_merge_log must link to one target voice_session
image_anchor_link_integrityimage anchor pointers must resolve to existing session messages
session_done_has_done_atvoice_session rows with is_active=false and to_finalize=true must have done_at
runtime_tag_coverage_voiceruntime-scoped voice collections should carry runtime_tag (legacy gaps are explicitly counted)
no_forecast_without_projecteach forecast_project_month must link to one project
type_normalized_idsall key attributes are non-empty strings

ID Normalization Strategy

Global strategy is stringify: BSON ObjectId to hex string, numeric ids to decimal strings, null/empty ids skipped and logged to deadletter.

bson.ObjectId → hex int/double → decimal string null/empty → skip + deadletter

Compose-Key Collections

Two collections use compose keys rather than single source fields. This is part of the canonical import contract.

forecasts_project_month: forecast_version_id + project_id + month finops_fx_rates: month + pair

Coalesce Rule in Scope

automation_tasks applies coalesced status mapping: task_status fallback to status. This is the only coalesce block declared in v1 mapping.

status := task_status || status

Runtime Coverage Focus

Voice runtime integrity is explicitly tracked in quality gates via runtime_tag_coverage_voice, matching runtime-scoped collections and legacy-gap accounting.

runtime_tag coverage check legacy gaps counted, not ignored
