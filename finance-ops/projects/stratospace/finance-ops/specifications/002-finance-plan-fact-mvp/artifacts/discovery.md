### 1) Резюме (1–3 строки)
Собираем требования к мини‑веб интерфейсу для руководителя, который показывает план‑факт и прогноз по проектам, управляет версиями прогноза, блокировками месяца, расчётами маржи и интегрируется с агентом K2 через запросы.

### 2) Бизнес-цели
- Цели: прозрачный план‑факт по проектам, контроль маржи/перерасхода, управляемый прогноз с версиями, управляемое закрытие периода.
- Критерий успеха: руководитель видит факт/прогноз/маржу по проектам в разрезе месяцев и может вносить правки/закрывать месяц с полным аудитом.
- Ограничения: MVP, фокус на горизонте года с рабочим фокусом 3 месяца; поддержка T&M и Fix.
- Definition of Done (простыми словами): план‑факт работает end‑to‑end (CRM → grid → edit → audit → lock), прогноз с версиями и копированием, базовая аналитика + alerts, маржа рассчитывается и объясняется.

### 3) Пользователи и текущий процесс
- Кто пользуется: админ/руководитель (полный доступ), агент K2 (техническая роль с жёстким audit‑log).
- Как делают сейчас: не описано в вводных (TBD).

### 4) Входящая информация (сырьё)
- `y-tasks-sandbox/other task/finance.md` — основной источник требований, расчётов, UX и структуры эпиков.
- CRM (источник данных): проекты, исполнители, часы (project/month/person/hours).
- ЦБ РФ (источник FX за месяц для USD→RUB).

### 5) Ключевые сценарии (кандидаты)
- Просмотр плана‑факта по проектам и месяцам → руководитель оценивает состояние → видит KPI и отклонения на главном экране.
- Редактирование факта/прогноза в Drawer → пересчёт сумм и FX → результат в grid + audit.
- Закрытие месяца (lock) → блокировка изменений факта → отображение статуса/замка.
- Управление версиями прогноза → переключение активной версии → сравнение с фактом.
- Копирование прогноза месяца вперёд → ускоряет планирование → результат в прогнозных ячейках.
- Запрос агенту K2 → формирование контекста и задачи → применение/предложение правок с audit.
- Аналитика и список внимания → выявление рисков/аномалий → переход к проекту/месяцу.

### 6) Master‑экраны (кандидаты)
- План‑факт (главный экран): KPI, таблица по клиентам/проектам/месяцам, фильтры, выбор версии прогноза, статус месяца.
- Drawer «Детали месяца»: вкладки Факт/Прогноз/Часы и маржа/История и файлы.
- Аналитика: KPI + «список внимания» + графики/таблицы драйверов.
- Настройки справочников: Клиенты/Проекты/Ставки/FX/Исполнители/Зарплаты (минимально по MVP).
- Модалки: Закрыть месяц, Копировать прогноз, Запрос агенту.
